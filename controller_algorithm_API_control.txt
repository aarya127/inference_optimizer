# PART 1 — Mathematical Definition of the Controller

---

# 1️⃣ Problem Setup

Each request ( i ) arrives with:

* Image resolution ( R_i )
* Prompt token count ( N_{p,i} )
* Estimated vision token count ( N_{v,i}(R) )

The system state at time ( t ):

[
\mathcal{S}_t =
{
B_t,,
Mem_t,,
KV_t,,
U_t,,
Frag_t
}
]

Where:

* ( B_t ) = active batch size
* ( Mem_t ) = memory used
* ( KV_t ) = total KV cache size
* ( U_t ) = GPU utilization
* ( Frag_t ) = fragmentation ratio

---

# 2️⃣ Strategy Space

Controller chooses:

[
a = (S_{parallel}, R, M_{kv}, B_{policy})
]

Where:

* ( S_{parallel} \in {single, replicate, TP} )
* ( R \in {224, 336, 448, 576, ...} )
* ( M_{kv} \in {contiguous, paged} )
* ( B_{policy} \in {static, continuous} )

This is your action space.

---

# 3️⃣ Cost Models

## Vision Stage

[
T_{vision}(S_{parallel}, R)
===========================

\frac{FLOPs(R)}{\Theta_{compute}}
+
\alpha \cdot N_{collectives}
+
\beta \cdot Bytes_{comm}
]

If replicate:
[
N_{collectives} = 1
]

If TP:
[
N_{collectives} \propto N_{blocks}
]

---

## Prefill Stage

Let:

[
N_{total} = N_v(R) + N_p
]

Then:

[
T_{prefill}(R) \approx
c_1 \cdot N_{total}^{\gamma}
]

Where:

* ( \gamma \in [1,2] ) depending on attention efficiency

---

## Decode Stage

KV size:

[
KV = B \cdot S \cdot L \cdot H
]

Per-token decode time:

[
T_{decode} \approx
\frac{Bytes_{weights} + Bytes_{KV}}{Bandwidth}
]

Paged allocation reduces effective memory waste:

[
KV_{effective} = KV \cdot (1 + Frag)
]

---

# 4️⃣ Total Latency

[
T_{total}(a)
============

T_{vision}
+
T_{prefill}
+
T_{decode}
]

---

# 5️⃣ Constraints

Memory constraint:

[
Mem_{weights}
+
KV_{effective}
\leq
GPU_{capacity}
]

SLA constraint:

[
TTFT \leq \tau
]

Optional accuracy constraint:

[
Accuracy(R) \geq \epsilon
]

---

# 6️⃣ Optimization Objective

Per-request:

[
a^* =
\arg\min_a T_{total}(a)
]

Subject to constraints.

---

# 7️⃣ Controller Algorithm (Deterministic Version)

At request arrival:

1. Extract features:
   [
   x = (N_v, N_p, B_t, Mem_t, Frag_t)
   ]

2. Enumerate feasible strategies ( a_j )

3. For each strategy:

   * Predict ( T_{total}(a_j) )
   * Check memory constraint

4. Select:
   [
   a^* = \min_j T_{total}(a_j)
   ]

5. Execute with configuration ( a^* )

This is a **model-predictive scheduler**.

---

# 8️⃣ Advanced Version (Online Learning)

You maintain:

[
\hat{T}_{model}
]

Update parameters via regression:

[
\theta \leftarrow \theta - \eta \nabla (T_{pred} - T_{actual})^2
]

Now it becomes adaptive.
